# 공공데이터 포털 크롤링 통합 Streamlit 앱 v5

## 변경 핵심

- 왼쪽 사이드바 탭 UI 적용
- 기관명 자동 보정/하드코딩 방식 제거
- 사용자가 입력한 기관명으로 포털 파일데이터 검색
- 검색 결과 카드에서 실제 `제공기관` 후보 추출
- 후보가 여러 개이면 사용자가 직접 선택
- 선택된 제공기관명으로 `org / orgFilter / orgFullName` URL 생성
- 크롤링 진행상황을 진행 바와 현재 단계 메시지로 실시간 표시
- 긴 URL/기관명/버튼 문구가 잘리지 않도록 CSS 수정

## 실행 방법

```bash
pip install -r requirements.txt
python -m playwright install chromium
streamlit run app.py
```

## 사용 흐름

1. 왼쪽 탭에서 기능 선택
2. 기관 검색어 입력
3. `제공기관 후보 조회` 클릭
4. 검색 결과에서 실제 수집할 제공기관 선택
5. 수집 시작
6. 진행 바와 로그 확인
7. 결과 파일 다운로드

## 기능 탭

- 전체 메타데이터
- 기관 메타데이터
- 조회·다운로드 집계
- 최신·과거 다운로드
- 실행 이력

## 기관명 처리 방식

이 버전은 `(주)`, `㈜`, `주식회사`, `강원특별자치도` 같은 기관명 변형을 코드가 임의로 맞히지 않습니다.

예를 들어 `강원도 고성군`으로 검색했을 때 포털 검색 결과에서 다음 후보가 나오면,

- 고성군
- 강원특별자치도 고성군
- 강원도 고성군
- 경상남도 고성군

앱이 자동으로 하나를 확정하지 않고 후보 목록을 보여줍니다. 사용자가 실제 수집할 제공기관을 직접 선택하면 그 선택값으로 최종 URL을 생성합니다.

## 산출물

실행 결과는 `runs/` 폴더 아래 작업별 폴더에 저장됩니다.

- 메타데이터 수집: `메타데이터.xlsx`, `실패로그.xlsx`
- 조회·다운로드 집계: `공공데이터_FILE_조회수_다운로드.xlsx`
- 파일 다운로드: 기관별 다운로드 폴더 및 ZIP
- 실행 로그: `run.log`
- 실행 결과 정보: `result.json`
